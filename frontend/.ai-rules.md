## 2. SOLID Principles (React Adaptation)

### Single Responsibility (SRP)
- 1 Component = 1 concern
- Pisahkan: **Page** (layout) в†’ **Component** (UI) в†’ **Hook** (logic) в†’ **Service** (API)
- Jangan campur fetch logic di component вЂ” gunakan custom hooks

### Open/Closed (OCP)
- Component harus extensible via props, bukan via modification
- Gunakan composition pattern (children, render props)

### Interface Segregation (ISP)
- Props harus spesifik вЂ” jangan passing entire object jika hanya butuh 2 field

### Dependency Inversion (DIP)
- Component depend on abstraction (hooks, context), bukan concrete API calls

## 3. React 19 Best Practices

### Component Pattern
```jsx
// вњ… Function component with named export
export default function GearCard({ name, price, image }) {
  return ( ... )
}

// вќЊ JANGAN class component
class GearCard extends React.Component { ... }
```

### State Management
```jsx
// вњ… useState untuk local state
const [isOpen, setIsOpen] = useState(false)

// вњ… useContext untuk shared state (auth, theme)
const { user, token } = useAuth()

// вќЊ JANGAN prop drilling lebih dari 2 level вЂ” gunakan Context
```

### Effect Best Practices
```jsx
// вњ… Cleanup effects
useEffect(() => {
  const controller = new AbortController()
  fetchData({ signal: controller.signal })
  return () => controller.abort()
}, [])

// вќЊ JANGAN fetch di useEffect tanpa cleanup
```

## 7. API Integration Pattern

```jsx
// вњ… Service layer pattern
// services/api.js
const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL,
  headers: { 'Content-Type': 'application/json' },
})

api.interceptors.request.use((config) => {
  const token = localStorage.getItem('token')
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})

// вњ… Feature-specific service
// features/auth/services/authService.js
export const login = (credentials) => api.post('/auth/login', credentials)
export const register = (data) => api.post('/auth/register', data)
```
